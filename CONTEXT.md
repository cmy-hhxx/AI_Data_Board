# AI Data Board — 领域术语

## 项目 (Project)

一个独立的工作单元，代表一件较大、有明确产出目标的事。例如：金融评测、工具评测、某批具体的图像数据标注。
- 每个项目有自己的任务集、可自定义的看板列和成员

## 看板列 (Board Column)

看板列定义任务在项目中的流转阶段。每个项目可以自定义自己的列。
- 创建项目时默认预设 4 列：待分配 → 进行中 → 紧急通道 → 已完成
- 列可以自由增、删、改名、重排序
- 列的顺序在 Swimlane 等视图中作为水平表头

## 任务 (Task)

项目中的最小工作单元，代表一件需要完成的事。
- 任务通过拖拽在看板列之间移动流转
- 每个任务有一个列内排序位置（position）
- 预估工天以「天」为单位（schema 字段名 `estimated_days`）
- 卡点（blocker）：任务无法推进的原因说明，供主管查看。当前阻塞状态字段

## 用户 (User) 与角色 (Role)

部门员工，是任务的执行人和负责人。
- supervisor — 主管，审阅整体进度
- pm — 项目经理，协调任务分配
- algorithm — 算法工程师，承担算法相关工作
- annotator — 标注员，执行数据标注
- crawler — 爬虫工程师，负责数据采集
- intern — 实习生
- 用户目前只读（种子数据），后续需增加增删改能力以应对人员变动


## 知识库 (Knowledge Base) 与 文档 (Document)

存放部门资料的模块，供全员阅读浏览，与看板项目无直接关联。
- 知识库是文档的容器，可排序
- 文档有名称、外链 URL、正文内容
- 未来方向：AI Native，即在知识库上集成 AI 能力

## 认证与权限

当前无登录/权限控制，所有用户平等访问。作为纯内部工具，暂无认证需求。

## 视图 (Views)

应用划分为 5 个视图，全员可访问，按场景选用：

| 视图                  | 位置                 | 用途                                         |
| --------------------- | -------------------- | -------------------------------------------- |
| 项目总览 (Overview)   | 首页 / 左上角 Logo   | 查看所有项目，选择进入；底部展示人员时间线   |
| 看板 (Board)          | 项目内默认           | 日常操作主界面，任务跨列拖拽流转             |
| 进度 (Progress)       | 项目内切换           | Swimlane 表（人×列矩阵），快速扫描负载与卡点 |
| 文档 (Document)       | 全局顶栏「文档」标签 | 查阅部门资料库                               |
| 人员时间线 (Timeline) | 项目总览页底部       | ECharts 甘特图，看所有人任务的时间分布       |

## 部署

- 当前仅本地开发环境运行（`pnpm dev`）
- 后续将部署到服务器

## 优先级 (Priority)

四档：low（低）、medium（中）、high（高）、urgent（紧急），各有对应颜色
